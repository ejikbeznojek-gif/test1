Обновление конфигурации через Интернет

Получение и установка обновления конфигурации могут быть выполнены программой автоматически. Обновление конфигурации включает три простых шага: подключение к сайту обновлений, получение обновления, создание резервной копии информационной базы (если необходимо) и обновление конфигурации.

Подключение к сайту обновлений

В некоторых организациях для соединения с Интернетом требуется указать имя пользователя и пароль для доступа на прокси-сервер. Для этого необходимо отметить флажок «Для доступа через прокси-сервер требуется указать имя пользователя и пароль прокси-сервера». Подробную информацию об использовании прокси-сервера можно получить у администратора вашей локальной сети.

Программа может при каждом запуске автоматически проверять наличие обновления конфигурации на пользовательском сайте. Для этого нужно установить флажок «Проверять наличие обновления конфигурации при каждом запуске программы». Если в результате проверки обновление будет обнаружено, программа предложит получить его с  пользовательского сайта и обновить конфигурацию.

Получение обновления

После нажатия  на кнопку «Далее» программа устанавливает соединение с пользовательским сайтом, используя указанные код пользователя и пароль, и производит поиск обновления для текущей конфигурации. Если такое обновление будет обнаружено, можно будет ознакомиться с его кратким описанием.

Имейте в виду, что программа может не всегда предлагать обновить конфигурацию сразу на самую последнюю версию. Это связано с тем, что каждое обновление подходит только для нескольких версий конфигурации. В этом случае программа поможет последовательно найти и установить обновления.

При следующем нажатии на кнопку «Далее» программа получит файл обновления, который будет сохранен в каталоге шаблонов. Получение файла обновления с пользовательского сайта может занять продолжительное время, в зависимости от объема конкретного файла и скорости подключения к Интернету.

Создание резервной копии информационной базы и обновление конфигурации

Перед началом обновления конфигурации необходимо создавать резервную копию информационной базы (путем копирования файла информационной базы или средствами SQL-сервера). Если резервная копия не была создана, программа может создать ее в режиме обновления конфигурации. Для этого необходимо установить флажок «Создавать резервную копию информационной базы» и указать папку, в которой будет сохранен файл выгрузки данных информационной базы.

Прежде чем продолжить обновлению конфигурации, необходимо выполнить следующие действия:

  • если в 1С:Предприятии есть несохраненные данные, сохранить их (или закрыть формы их редактирования);
  • если открыт Конфигуратор, закрыть его;
  • если с информационной базой работают другие пользователи, предупредить их о необходимости временно прекратить работу;
  • если к информационной базе сейчас подключены другие программы в режиме внешнего соединения, необходимо завершить их работу.


Если все указанные условия выполнены, после нажатия на кнопку «Далее» программа автоматически создаст резервную копию информационной базы (при условии, что установлен соответствующий флаг) и обновит конфигурацию. На время создания резервной копии и обновления конфигурации 1С:Предприятие будет закрыто. Процесс обновления конфигурации может занять продолжительное время, в зависимости от производительности компьютера и объема информационной базы.

После завершения обновления конфигурации 1С:Предприятие будет запущено автоматически (при этом, возможно, потребуется повторно указать имя пользователя и пароль для входа в систему). При первом запуске обновленной конфигурации будет выполнено обновление информационной базы. После этого программа будет готова к дальнейшей работе.

Режим обновления конфигурации из 1С:Предприятия включает ряд ограничений. Он не может быть использован при работе с операционной системой Windows 98, а также в том случае, если в конфигурации включена возможность изменения объектов метаданных. В таких ситуациях рекомендуется осуществлять обновление при помощи Конфигуратора (меню «Конфигурация» - «Поддержка» - «Обновить конфигурацию»).
