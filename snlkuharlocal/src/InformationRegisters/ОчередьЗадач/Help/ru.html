## Описание логики работы регистра "ОчередьЗадач"

### Назначение
Регистр "ОчередьЗадач" предназначен для управления асинхронным выполнением задач через фоновые задания в 1С:Предприятие. Он обеспечивает очередность выполнения, контроль состояния и мониторинг задач.

### Структура регистра

**Измерения:**
- `ИД_Задачи` (UUID) - уникальный идентификатор задачи
- `НомерЧасти` (Число) - номер части для разбивки крупных задач на части

**Основные ресурсы:**
- `ИмяОчереди` - наименование очереди задач
- `ИмяМетода` - имя метода конфигурации для выполнения
- `Приоритет` - приоритет задачи (меньше число = выше приоритет)
- `Основание` - объект-основание для задачи
- `Автор` - пользователь, создавший задачу
- `СостояниеЗадания` - текущее состояние (перечисление СостоянияФЗ)
- `ИсполняющееФЗ` - UUID фонового задания
- `пОбработки` - параметры обработки в ХранилищеЗначения
- `ДатаЗапуска_План` - плановая дата запуска
- `ДатаЗапуска_Факт` - фактическая дата запуска
- `ДатаЗавершения` - дата завершения
- `Успешно` - признак успешного выполнения
- `Ошибки` - текст ошибок при выполнении
- `ВремяВыполнения` - время выполнения в секундах
- `КоличествоОбъектов` - количество обработанных объектов
- `Процесс` - ссылка на процесс обработки
- `МаксЧислоПопыток` - максимальное число попыток
- `НомерПопытки` - текущий номер попытки

### Основные функции

#### 1. СоздатьЗадачу()
Создает новую задачу в очереди с проверкой на дубликаты:
- Проверяет наличие задачи с тем же методом и основанием
- Если задача существует и не указан параметр "Повторно" - возвращает ошибку
- Создает запись с уникальным ID и текущей датой запуска
- Сохраняет параметры обработки в сжатом виде

#### 2. ЗапуститьЗадачу()
Запускает выполнение задачи через фоновое задание:
- Проверяет текущее состояние задачи (не должно быть запущено)
- Извлекает параметры обработки из ХранилищеЗначения
- Создает фоновое задание через `ФоновыеЗадания.Выполнить()`
- Обновляет статус задачи на "Активно"
- Фиксирует дату фактического запуска и ID фонового задания

#### 3. НайтиЗадачу()
Ищет задачу по методу, основанию и номеру части:
- Возвращает задачи в статусах "ПустаяСсылка" или "Активно"
- Используется для предотвращения дублирования задач

#### 4. ОбслужитьОчереди()
Главная функция обслуживания всех очередей:
- Запускает планировщик отложенной обработки
- Обновляет статусы активных задач
- Запускает новые задачи из очереди с учетом лимитов

#### 5. ЗапуститьЗадачи()
Запускает задачи из очереди с учетом приоритетов:
- Выбирает задачи с плановой датой запуска <= текущей дате
- Сортирует по имени очереди, приоритету и дате запуска
- Группирует по именам очередей
- Соблюдает лимит одновременных задач (максимум 10)

#### 6. ОбновитьСтатусыЗадач()
Мониторинг состояния активных задач:
- Выбирает все задачи в статусе "Активно"
- Проверяет реальное состояние фоновых заданий
- Обновляет статусы завершенных/аварийно завершенных задач
- Возвращает количество активных задач по очередям

#### 7. ОбновитьСостояниеИсполняющегоФЗ()
Обновляет состояние конкретной задачи:
- Устанавливает статус в зависимости от состояния ФЗ
- Фиксирует результаты выполнения (успешно/ошибки)
- Записывает дату завершения

### Жизненный цикл задачи

1. **Создание:** `СоздатьЗадачу()` → задача со статусом "ПустаяСсылка"
2. **Запуск:** `ЗапуститьЗадачу()` → статус "Активно", создается ФЗ
3. **Выполнение:** фоновое задание выполняет метод конфигурации
4. **Завершение:** `ОбновитьСостояниеИсполняющегоФЗ()` → статус "Завершено" или "ЗавершеноАварийно"
5. **Мониторинг:** `ОбновитьСтатусыЗадач()` проверяет состояние ФЗ

### Особенности

- **Поддержка частей:** задачи могут разбиваться на части для обработки больших объемов данных
- **Приоритизация:** задачи выполняются по приоритету (меньше число = выше приоритет)
- **Лимиты:** максимальное количество одновременно выполняемых задач в очереди - 10
- **Обработка ошибок:** фиксируются ошибки и время выполнения
- **Повторные запуски:** поддерживается параметр "Повторно" для перезапуска задач
